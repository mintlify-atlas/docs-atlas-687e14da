<Tabs>
  <Tab title="Docker (recommended)">
    <Steps>
      <Step title="Pull the image">
        Pull the latest RadishDB image from Docker Hub:

        ```bash
        docker pull piee314/radishdb:latest
        ```
      </Step>

      <Step title="Start the container">
        Run RadishDB with a persistent volume and exposed port:

        ```bash
        docker run -d \
          --name radishdb \
          -p 6379:6379 \
          -v radish-data:/app/aof \
          piee314/radishdb:latest
        ```

        This starts RadishDB in TCP server mode on port `6379`. The `-v` flag mounts a named volume so your data survives container restarts.

        <Note>
          The AOF (append-only file) is written to `/app/aof/radish.aof` inside the container. Mounting the `radish-data` volume at `/app/aof` ensures your data persists across restarts and container replacements. Without this volume, all data is lost when the container stops.
        </Note>
      </Step>

      <Step title="Connect with netcat">
        Once the container is running, open a connection to the server:

        ```bash
        nc localhost 6379
        ```

        You are now connected to the RadishDB TCP server and can send commands directly.
      </Step>

      <Step title="Run your first commands">
        Try the following session to verify everything works:

        ```text
        SET name radish
        OK

        GET name
        radish

        SET color red EX 10
        OK

        TTL color
        7

        INFO
        Version : 0.1
        Uptime Seconds : 12

        Keys : 2
        Keys with TTL : 1
        Expired keys : 0

        Buckets : 8
        Load Factor : 0.250000
        Resizes : 0
        Max chain : 1
        ```

        `SET name radish` stores a key without expiration. `SET color red EX 10` stores a key that expires in 10 seconds. `TTL color` returns the remaining seconds. `INFO` shows live engine statistics.
      </Step>
    </Steps>
  </Tab>

  <Tab title="Native build">
    <Steps>
      <Step title="Prerequisites">
        You need `gcc` and `make` installed. On Debian/Ubuntu:

        ```bash
        sudo apt-get install gcc make
        ```

        On macOS with Homebrew:

        ```bash
        brew install gcc make
        ```
      </Step>

      <Step title="Clone and build">
        Clone the repository and compile with `make`:

        <CodeGroup>

        ```bash clone
        git clone https://github.com/piee314/radishdb.git
        cd radishdb
        ```

        ```bash build
        make
        ```

        </CodeGroup>

        The Makefile compiles all source files under `src/` using `gcc` with `-Wall -Wextra -Werror` and outputs the `radishdb` binary in the project root.
      </Step>

      <Step title="Choose a run mode">
        RadishDB supports two modes:

        <CodeGroup>

        ```bash REPL mode
        ./radishdb
        ```

        ```bash TCP server mode
        ./radishdb --server
        ```

        </CodeGroup>

        **REPL mode** opens an interactive prompt in your terminal. **Server mode** listens on port `6379` and accepts connections from any TCP client, including `nc`.

        <Tip>
          Use REPL mode for local exploration. Use `--server` (or `-s`) when you want to connect from another process or test network behaviour.
        </Tip>
      </Step>

      <Step title="Run your first commands">
        In REPL mode, type commands at the prompt. In server mode, connect first:

        ```bash
        nc localhost 6379
        ```

        Then run the same session:

        ```text
        SET name radish
        OK

        GET name
        radish

        SET color red EX 10
        OK

        TTL color
        7

        INFO
        Version : 0.1
        Uptime Seconds : 5

        Keys : 2
        Keys with TTL : 1
        Expired keys : 0

        Buckets : 8
        Load Factor : 0.250000
        Resizes : 0
        Max chain : 1
        ```

        AOF is written to `aof/radish.aof` relative to the working directory. On the next start, RadishDB replays the log automatically so your keys survive restarts.
      </Step>
    </Steps>
  </Tab>
</Tabs>

## What's next

Now that RadishDB is running, explore what it can do:

- Run `HELP` at any prompt to see all supported commands.
- Use `DEL key` to remove a key, `COUNT` to see how many keys are stored.
- Use `SAVE mysnap` to write a binary `.rdbx` snapshot and `LOAD mysnap` to restore it.
- Run `BENCH 10000` to insert 10,000 keys and measure write throughput.
